# Vision Template

Use this template when generating VISION.md. Fill all sections from Stages 1-2 questioning data.

## Template

```markdown
# Product Vision: [Product Name]

## Elevator Pitch
For [target users] who [statement of need],
[Product Name] is a [product category] that [key benefit].
Unlike [primary alternative], our product [primary differentiation].

## Problem
What problem exists in the world that this product addresses?
[2-3 sentences — the pain, not the solution]

## Target Users
| User Type | Description | Primary Need |
|-----------|-------------|-------------|
| ...       | ...         | ...         |

## Value Proposition
What value does this product deliver? Why would someone choose it?
[2-3 sentences — outcomes, not features]

## Differentiation
What makes this product uniquely positioned?
- [Differentiator 1]
- [Differentiator 2]
- [Differentiator 3]

## Success Vision (3-5 Year Horizon)
What does the world look like if this product succeeds?
[2-3 sentences — the aspirational future state]
```

## Field Guidance

| Field | Source | Notes |
|-------|--------|-------|
| Elevator Pitch | Geoffrey Moore format from Crossing the Chasm | All brackets must be filled — no placeholders |
| Problem | Stage 1 questioning | Pain statement, not solution description |
| Target Users | Stage 2 questioning | Roles and behaviors, not demographics |
| Value Proposition | Stage 2 questioning | Outcomes delivered, not feature list |
| Differentiation | Stages 1-2 | Brief bullets, not competitive analysis |
| Success Vision | Stage 1 questioning | Aspirational 3-5 year horizon |

## Constraints

- **1 page maximum** — Vision is a reference document. If it's long, nobody reads it.
- **No metrics** — Metrics belong in PRD §4 (Goals & Success Metrics)
- **No roadmap/timeline** — That's ROADMAP.md
- **No competitive analysis** — Brief differentiation bullets suffice
